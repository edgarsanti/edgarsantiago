Introducción¶

La interseccin de listas ordenadas es una operacin central en el modelo de comparacin y en tareas de recuperacin de informacin, donde cada lista representa un conjunto de documentos o registros que cumplen cierta condicin (Baeza-Yates & Ribeiro-Neto, 2011). En este contexto, interesa no solo obtener correctamente la interseccin, sino tambin minimizar el tiempo de ejecucin y el nmero de comparaciones entre claves.

En este reporte se estudian tres esquemas de interseccin de conjuntos: Melding (ME), que intersecta primero las listas ms pequeas para reducir el tamao de los resultados intermedios; BaezaYates (BY), un algoritmo divide-y-vencers que utiliza una lista gua y realiza bsquedas sobre las dems; y Barbay & Kenyon (BK), que mantiene punteros en todas las listas y actualiza de forma adaptativa un candidato comn. El algoritmo de BaezaYates se implementa parametrizado con tres estrategias de bsqueda sobre listas ordenadas: bsqueda binaria clsica y dos variantes de bsqueda no acotada (B1 y B2).

Para la evaluacin emprica se utilizan tres familias de datos (A, B y C), correspondientes a pares, tripletas y tetrapletas de listas ordenadas. En cada familia se mide, para cada algoritmo, el tiempo de ejecucin, el nmero de comparaciones y la longitud de la interseccin resultante, repitiendo los experimentos varias veces y resumiendo los resultados mediante tablas y grficos tipo boxplot.

Limpieza de datos¶

Antes de aplicar los algoritmos de interseccin, se realiza una limpieza de los datos para que cada lista se comporte realmente como un conjunto ordenado. Para ello, se define una funcin que recorre cada grupo de la familia (A, B o C) y, dentro de cada grupo, procesa cada lista de identificadores.

En cada lista se aplica primero set(L) para eliminar elementos duplicados y despus sorted(...) para ordenar los valores de menor a mayor. El resultado es una nueva estructura en la que todas las listas estn libres de repeticiones y en orden creciente, lo que coincide con el supuesto terico de conjuntos ordenados en el modelo de comparacin.

Se decidi hacer esta limpieza porque, en los archivos originales, pueden aparecer valores repetidos o desordenados que no forman parte del modelo ideal. Al normalizar, garantizamos que los datos cumplen las condiciones tericas, evitamos comparaciones extra debidas a ruido en los datos y obtenemos resultados ms claros y consistentes en los experimentos. A partir de esta normalizacin se construyen las versiones A, B y C que se utilizan en las pruebas.

def normalizar_familia(F):
    F_norm = []
    for grupo in F:

        for L in grupo:

            nuevo_grupo.append(L_norm)

Contador de comparaciones y medición de tiempo¶

Para evaluar el desempeo de los algoritmos no solo se considera el tiempo de ejecucin, sino tambin el nmero de comparaciones realizadas entre claves. La clase CmpCounter se encarga de llevar ese conteo: inicializa un contador en cero y define tres mtodos (lt, le y eq) que encapsulan las comparaciones <, <= y ==. Cada vez que uno de estos mtodos se llama dentro de un algoritmo, se incrementa el contador y se devuelve el resultado lgico de la comparacin. De esta forma, al final de cada experimento se puede saber cuntas comparaciones realiz el algoritmo sobre las listas.

Adems, se define la funcin medir_tiempo, que recibe una funcin y sus argumentos, ejecuta el cdigo y mide el intervalo de tiempo usando time.perf_counter(). Esta funcin devuelve tanto el resultado de la operacin como el tiempo transcurrido en segundos. Combinando CmpCounter y medir_tiempo es posible registrar, para cada algoritmo y conjunto de listas, dos mtricas clave: el nmero de comparaciones y el tiempo de ejecucin.

In [7]:
class CmpCounter:
    def __init__(self):
        self.count = 0
    def lt(self, a, b):
        self.count += 1
        return a < b
    def le(self, a, b):
        self.count += 1
        return a <= b
    def eq(self, a, b):
        self.count += 1
        return a == b
def medir_tiempo(fn, *args, **kwargs):
    t0 = time.perf_counter()
    res = fn(*args, **kwargs)
    t1 = time.perf_counter()
    return res, (t1 - t0)

Algoritmos de búsqueda: binaria, B1 y B2¶

En esta seccin se definen tres rutinas de bsqueda sobre listas ordenadas que se utilizan como subrutinas dentro de los algoritmos de interseccin. Todas reciben una lista A, un valor objetivo x, una posicin inicial lo y el contador de comparaciones cmp, siguiendo el enfoque del modelo de comparacin presentado en los materiales del curso (Sadit, s. f.-a; Baeza-Yates & Ribeiro-Neto, 2011).

La funcin findpos_binaria implementa una bsqueda binaria clsica. A partir del rango [lo, len(A)) va partiendo el segmento a la mitad hasta localizar la primera posicin p donde A[p] es mayor o igual que x (lower_bound). Cada comparacin se realiza a travs de cmp.lt, de modo que el nmero de comparaciones queda registrado.

La funcin findpos_B1 implementa una bsqueda no acotada de tipo doubling search. Primero avanza con saltos crecientes (1, 2, 4, 8, ) desde la posicin lo hasta encontrar un rango donde puede estar x. Una vez acotado ese intervalo, aplica bsqueda binaria dentro de ese subrango. Esta estrategia puede ser ms eficiente cuando la posicin de x est relativamente cerca del punto de partida.

La funcin findpos_B2 tambin realiza una bsqueda no acotada, pero con saltos que crecen ms rpido (2, 4, 16, 256, ), siguiendo una idea de double-doubling. Igual que en B1, al encontrar un intervalo que contiene a x se realiza una bsqueda binaria final. En ambos casos, B1 y B2 usan cmp.lt para contabilizar las comparaciones y permiten estudiar cmo cambia el rendimiento de los algoritmos de interseccin al variar la estrategia de bsqueda.

def findpos_binaria(A, x, lo, cmp: CmpCounter):
    hi = len(A)
    while lo < hi:
        mid = (lo + hi) // 2
        if cmp.lt(A[mid], x):
            lo = mid + 1
        else:
            hi = mid
    return lo

def findpos_B1(A, x, lo, cmp: CmpCounter):
    n = len(A)
    if lo >= n:
        return n
    if not cmp.lt(A[lo], x):
        return lo
    bound = 1
    while lo + bound < n:
        if cmp.lt(A[lo + bound], x):
            bound *= 2
        else:
            break
    left = lo + bound // 2
    right = min(lo + bound, n)
    if left < lo:
        left = lo

    while left < right:
        mid = (left + right) // 2
        if cmp.lt(A[mid], x):
            left = mid + 1
        else:
            right = mid
    return left

def findpos_B2(A, x, lo, cmp: CmpCounter):
    n = len(A)
    if lo >= n:
        return n
    if not cmp.lt(A[lo], x):
        return lo
    step = 2
    prev = 0
    while True:
        idx = lo + step
        if idx >= n:
            left = lo + prev
            right = n
            break
        if cmp.lt(A[idx], x):
            prev = step
            step = step * step
        else:
            left = lo + prev
            right = idx
            break

    if left >= n:
        return n

    while left < right:
        mid = (left + right) // 2
        if cmp.lt(A[mid], x):
            left = mid + 1
        else:
            right = mid
    return left

Intersección de dos listas con el algoritmo de Baeza–Yates¶

La funcin by_intersection_2 implementa el algoritmo de BaezaYates para calcular la interseccin de dos listas ordenadas A y B. La funcin recibe tambin un procedimiento de bsqueda findpos (puede ser binaria, B1 o B2) y el contador de comparaciones cmp. Los parmetros a_lo, a_hi, b_lo y b_hi delimitan los rangos activos de A y B, y permiten que el algoritmo se llame de forma recursiva sobre subsegmentos, siguiendo la idea presentada por Baeza-Yates en el contexto de recuperacin de informacin (Baeza-Yates & Ribeiro-Neto, 2011; Sadit, s. f.-b).

La idea principal es de tipo divide y vencers. En cada llamada se toma un elemento pivote median en la mitad del segmento de A. Luego se busca esa mediana en B mediante findpos. Si la posicin encontrada queda fuera del rango activo de B, se contina recursivamente solo en la parte izquierda de A, porque la interseccin solo puede estar all. Si se encuentra una coincidencia exacta (cmp.eq), primero se exploran recursivamente los subrangos de la izquierda, despus se agrega la mediana a la salida y finalmente se procesan los subrangos de la derecha. Si no hay coincidencia, se dividen los rangos en izquierda y derecha sin agregar nada a la salida. De esta forma, by_intersection_2 construye recursivamente la interseccin A B utilizando la rutina de bsqueda que se le pasa como parmetro.

In [9]:
def by_intersection_2(A, B, findpos, cmp: CmpCounter,
                      a_lo=0, a_hi=None, b_lo=0, b_hi=None, out=None):
    if a_hi is None:
        a_hi = len(A)
    if b_hi is None:
        b_hi = len(B)
    if out is None:
        out = []
    if a_lo >= a_hi or b_lo >= b_hi:
        return out
    mid = (a_lo + a_hi) // 2
    median = A[mid]
    pos = findpos(B, median, b_lo, cmp)
    if pos >= b_hi:
        return by_intersection_2(A, B, findpos, cmp,
                                 a_lo, mid, b_lo, b_hi, out)
    if cmp.eq(B[pos], median):
        by_intersection_2(A, B, findpos, cmp,
                          a_lo, mid, b_lo, pos, out)
        out.append(median)
        by_intersection_2(A, B, findpos, cmp,
                          mid + 1, a_hi, pos + 1, b_hi, out)
    else:
        by_intersection_2(A, B, findpos, cmp,
                          a_lo, mid, b_lo, pos, out)
        by_intersection_2(A, B, findpos, cmp,
                          mid + 1, a_hi, pos, b_hi, out)
    return out

Variantes de Baeza–Yates según la búsqueda utilizada¶

Estas funciones definen tres variantes del algoritmo de BaezaYates para la interseccin de dos listas, segn el tipo de bsqueda que se utilice internamente. La funcin BY_binaria llama a by_intersection_2 usando findpos_binaria, es decir, emplea bsqueda binaria clsica para localizar cada pivote en la segunda lista. La funcin BY_B1 hace lo mismo pero usando findpos_B1, que implementa una bsqueda no acotada con saltos crecientes antes de aplicar una bsqueda binaria final. Finalmente, BY_B2 utiliza findpos_B2, que realiza una bsqueda no acotada con saltos que crecen ms rpidamente. Estas tres variantes permiten comparar el efecto de la estrategia de bsqueda sobre el comportamiento del algoritmo de BaezaYates.

Intersección de varias listas con Baeza–Yates¶

La funcin BY_multi extiende el algoritmo de BaezaYates al caso de k listas. Recibe una lista de listas lists, una funcin base_BY que calcula la interseccin de dos listas (por ejemplo, BY_binaria, BY_B1 o BY_B2) y el contador de comparaciones cmp. El procedimiento toma la primera lista como resultado inicial y luego, de manera secuencial, la va intersectando con cada una de las listas restantes. Si en algn punto la interseccin parcial queda vaca, el ciclo se detiene porque ya no puede aparecer ningn elemento en comn.

A partir de esta funcin general se definen tres variantes especficas: BY_multi_binaria, que utiliza BY_binaria como base; BY_multi_B1, que utiliza BY_B1; y BY_multi_B2, que utiliza BY_B2. De este modo se obtiene la interseccin de k listas usando BaezaYates parametrizado con bsqueda binaria, con B1 o con B2, lo que permite comparar estas tres opciones en los experimentos.

In [11]:
def BY_multi(lists, base_BY, cmp: CmpCounter):
    if not lists:
        return []
    res = lists[0]
    for nxt in lists[1:]:
        res = base_BY(res, nxt, cmp)
        if not res:
            break
    return res
def BY_multi_binaria(lists, cmp):
    return BY_multi(lists, BY_binaria, cmp)
def BY_multi_B1(lists, cmp):
    return BY_multi(lists, BY_B1, cmp)
def BY_multi_B2(lists, cmp):
    return BY_multi(lists, BY_B2, cmp)

Intersección Melding (ME) con estrategia small-vs-small¶

La funcin ME_melding implementa la estrategia Melding (small-vs-small) para la interseccin de varias listas. Primero ordena las listas de entrada segn su longitud, de menor a mayor. Luego toma la lista ms pequea como resultado inicial y la va intersectando, una por una, con las listas restantes mediante la funcin base_intersection, que calcula la interseccin de dos listas. Si en algn punto la interseccin parcial queda vaca, el ciclo se detiene porque ya no puede aparecer ningn elemento en comn, siguiendo la idea de privilegiar las listas ms cortas discutida en el contexto de interseccin adaptativa (Barbay & Kenyon, 2002; Barbay, Lpez-Ortiz, & Lu, 2006; Sadit, s. f.-b).

La funcin ME_BY_binaria es una instancia concreta de este esquema, donde la interseccin de dos listas se realiza con BY_binaria. De esta forma, ME utiliza BaezaYates con bsqueda binaria como rutina base, pero mantiene la idea central de intersectar primero las listas ms pequeas para reducir el tamao de los resultados intermedios.

In [12]:
def ME_melding(lists, base_intersection, cmp: CmpCounter):
    if not lists:
        return []
    work = sorted(lists, key=len)
    curr = work[0]
    for nxt in work[1:]:
        curr = base_intersection(curr, nxt, cmp)
        if not curr:
            break
    return curr
def ME_BY_binaria(lists, cmp):
    return ME_melding(lists, BY_binaria, cmp)

Intersección de varias listas con el algoritmo de Barbay y Kenyon (BK)¶

La funcin BK_intersection implementa el algoritmo de Barbay y Kenyon para calcular la interseccin de k listas ordenadas. En primer lugar se revisa si alguna lista est vaca; en ese caso, la interseccin es vaca. Tambin se consideran los casos triviales de cero listas (devuelve una lista vaca) y de una sola lista (devuelve una copia de esa lista). Para el caso general, se inicializa un arreglo de punteros P, uno por cada lista, y se toma como candidato inicial el el primer elemento de la primera lista.

Dentro del ciclo principal, el algoritmo intenta alinear el candidato el en todas las listas. Para cada lista i, se busca la posicin adecuada mediante findpos, empezando desde el puntero actual P[i]. Si la posicin queda fuera de la lista, se termina y se devuelve la interseccin acumulada. En caso contrario, se actualiza P[i] y se compara el valor encontrado con el candidato. Si todas las listas coinciden en el mismo valor, se agrega ese elemento a la salida y se avanza el puntero de la primera lista para buscar el siguiente candidato. Si alguna lista tiene un valor mayor, se actualiza el a ese nuevo valor y se reinicia el conteo de coincidencias. De esta forma, BK recorre las listas de manera coordinada, usando findpos para avanzar, y construye la interseccin paso a paso. La funcin BK_binaria fija que las bsquedas internas se hagan con findpos_binaria, es decir, con bsqueda binaria sobre cada lista.

In [13]:
def BK_intersection(lists, findpos, cmp: CmpCounter):
    if any(len(lst) == 0 for lst in lists):
        return []
    L = lists
    k = len(L)
    if k == 0:
        return []
    if k == 1:
        return L[0][:]
    P = [0] * k
    out = []
    el = L[0][0]
    while True:
        c = 0
        for i in range(k):
            pos = findpos(L[i], el, P[i], cmp)
            if pos >= len(L[i]):
                return out
            P[i] = pos
            val = L[i][pos]
            if cmp.eq(val, el):
                c += 1
            else:
                el = val
                c = 1
        if c == k:
            out.append(el)
            P[0] += 1
            if P[0] >= len(L[0]):
                return out
            el = L[0][P[0]]
def BK_binaria(lists, cmp):
    return BK_intersection(lists, findpos_binaria, cmp)

Ejecución de un experimento¶

La funcin correr_experimento ejecuta varias veces el mismo algoritmo de interseccin sobre un grupo de listas y registra las mtricas obtenidas. En cada repeticin se crea un nuevo contador CmpCounter, se llama a algofn usando medir_tiempo para obtener la interseccin y el tiempo de ejecucin, y se guarda una fila con la familia, el identificador de grupo, el nmero de listas k, el nombre del algoritmo, el ndice de repeticin, el tiempo en segundos, el nmero de comparaciones y la longitud de la interseccin. Al final, todas las repeticiones se devuelven como un DataFrame de pandas.

En este caso se utiliza reps = 100 para cada combinacin de familia, grupo y algoritmo. Hacer muchas repeticiones ayuda a reducir el efecto del ruido en la medicin del tiempo (variaciones del sistema, carga de la mquina, etc.) y permite estimar de manera ms estable los tiempos promedio y la variabilidad de cada algoritmo. As, los resultados que se analizan en las tablas y grficas son ms confiables y menos sensibles a una sola ejecucin atpica.

In [27]:
def correr_experimento(familia, grupo_id, listas, algoname, algofn, reps=100):
    filas = []
    for r in range(reps):
        cmp = CmpCounter()
        inter, dt = medir_tiempo(algofn, listas, cmp)
        filas.append({
            "familia": familia,
            "grupo": grupo_id,
            "k": len(listas),
            "algoritmo": algoname,
            "rep": r,
            "tiempo_s": dt,
            "comparaciones": cmp.count,
            "len_inter": len(inter),
        })
    return pd.DataFrame(filas)

Bucle principal de experimentos y construcción del DataFrame global¶

La funcin correr_todos ejecuta el experimento completo sobre las tres familias de datos. Recorre en orden las familias A, B y C, y para cada una itera sobre todos sus grupos mediante enumerate, obteniendo el identificador de grupo (gid) y las listas asociadas. Para cada grupo, aplica todos los algoritmos definidos en ALGORITMOS_5, llamando a correr_experimento con la familia, el grupo, las listas, el nombre del algoritmo y el nmero de repeticiones reps (en este caso, 100). Cada llamada devuelve un DataFrame con las repeticiones de esa combinacin, que se acumula en la lista dfs. Al final, todos estos resultados se concatenan con pd.concat para formar un solo DataFrame global (df_res), que contiene la informacin de todas las familias, grupos, algoritmos y repeticiones y sirve como base para los resmenes y los grficos de comparacin.

def correr_todos(A, B, C, reps=100):
    dfs = []
    for familia, F in [("A", A), ("B", B), ("C", C)]:
        for gid, listas in enumerate(F):
            for algoname, algofn in ALGORITMOS_5.items():
                dfs.append(
                    correr_experimento(familia, gid, listas,
                                       algoname, algofn, reps))
    return pd.concat(dfs, ignore_index=True)

Conclusiones¶

En este trabajo se compararon cinco algoritmos de interseccin (ME, BK y tres variantes de BaezaYates con bsqueda binaria, B1 y B2) sobre tres familias de listas (A, B y C). Para cada combinacin se realizaron 100 repeticiones y se midieron tiempo de ejecucin y nmero de comparaciones en el modelo de comparacin.

Los resultados son consistentes en todas las familias: ME, basado en la estrategia small-vs-small, es siempre el algoritmo ms eficiente, con los menores tiempos medios y el menor nmero de comparaciones. BK aparece como una alternativa intermedia razonable, mientras que las variantes de BaezaYates son claramente ms costosas; BY_binaria es la peor, y B1/B2 mejoran su comportamiento pero sin alcanzar a ME ni a BK.

Finalmente, el tamao de la interseccin fue el mismo para todos los algoritmos dentro de cada familia, por lo que las diferencias observadas se deben nicamente a la forma en que cada mtodo organiza las bsquedas y las comparaciones. En este escenario, ME se perfila como la opcin preferente para intersectar listas tipo posting lists, con BK como segunda opcin y las variantes de BY principalmente como referencia para estudiar el efecto de distintas estrategias de bsqueda.
